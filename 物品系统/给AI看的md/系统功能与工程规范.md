# 商店背包系统 · 功能清单与工程规范

> 设计背景：[[商店背包系统]]（部分与实现有 drift，**以本文为准**）。  
> 本文描述**当前 Unity 工程已实现的功能**及**不可随意破坏的行为约束**。

---

## §1 系统边界

| 本系统负责 | 本系统不负责 |
|---|---|
| 物品 SO 定义、背包/商店/装备/货币**运行时数据** | 战斗逻辑、GameplayEffect **执行** |
| 买/卖/装/卸/消耗品扣数量 | 角色属性**最终结算**（只提供 `GetAllStatMods` 等查询） |
| UI：商店、背包、信息栏 | 正式 HUD 入口（测试用 B/I；商店接入总工程后改外部触发） |
| 存档块 JSON 捕获/还原（内存） | 写磁盘 / PlayerPrefs（留给主工程存档系统） |

**依赖方向**：`UI → Service → Data`；禁止反向。

---

## §2 功能模块与细节要求

### 2.1 物品数据（Data/Item）

| 功能 | 细节要求 |
|---|---|
| 三子类 | `Equipment` / `Consumable` / `StoryItem`，继承 `ItemBase` |
| id 规则 | 策划填 `idSuffix`；`OnValidate` 写只读 `id = prefix + idSuffix` |
| 资产重命名 | `id + "_" + Name` 自动重命名（Editor） |
| 堆叠 | 消耗品按 `maxStack`；装备/剧情 `count=1` + 装备独立 `instanceGuid` |
| 前缀 | `equip_` / `cons_` / `story_`（不合规 LogWarning） |

### 2.2 货币（WalletService）

| 功能 | 细节要求 |
|---|---|
| 本期币种 | 仅 UI 显示 `Ink`；`Spirit` 预留 |
| API | `Get` / `Has` / `Add` / `TrySpend` |
| 原子性 | `TrySpend` 不足时不扣 |
| 上限 | C# `int.MaxValue` |
| 事件 | `OnChanged(CurrencyId, int)` |

### 2.3 背包（Inventory）

| 功能 | 细节要求 |
|---|---|
| 容量 | **无限**（无拒绝加物） |
| 加物 | `TryAdd(item, count)` — 按类别堆叠或新格 |
| 取物 | `TryTakeAt` / `TryTakePartialAt` / `TryAddStack`（卸装备回包） |
| 消耗 | `TryConsume(id, count)` 跨格扣；`TryConsumeAt(index, count, UseContext, out effects)` 按格 + 上下文 |
| 查询 | `GetConsumables(UseContext?)` |
| 事件 | `OnChanged` → UI 刷新 |
| 存档 | `Data` / `LoadFromData`；`capacity=0` 表示无限 |

### 2.4 商店（ShopService + ShopTableSO）

| 功能 | 细节要求 |
|---|---|
| 配置 | `ShopTableSO`：`idSuffix` → `shopId=shop_+suffix`；资产自动重命名 |
| 开店 | `Open(table)` 建 `entryById` 索引；恢复已归档 `ShopRuntimeData` |
| 关店 | `Close()` **归档**当前 runtime（限购剩余） |
| 购买 | `TryBuy`：校验库存 → 扣 Ink → `TryAdd` → 扣装备库存；失败回滚 Ink |
| 卖出 | `TrySell(bagIndex, count)`：`basePrice` × count；`canSell=false` 拒绝 |
| 库存规则 | 消耗品/剧情：**忽略** stock；符文：`stock>0` 限购，`-1` 无限，`0` 不卖 |
| 单价 | `(priceOverride>0 ? override : basePrice) × priceMultiplier` |

### 2.5 符文装备（EquipmentService）

| 功能 | 细节要求 |
|---|---|
| 槽位 | N 个**同质**符文槽（默认 3），`slotIndex` 0…N-1 |
| 装/卸 | 背包移除 → 离包缓存；卸下 → `TryAddStack` 回包（无限背包必成功） |
| 换装 | 同槽先卸旧件；按 **guid** 找回背包格（防 index 偏移） |
| 查询 | `GetEquippedStack/Item`；`GetAllStatMods/SkillMods/ExtraEffects`（聚合已装符文） |
| 事件 | `OnEquipped` / `OnUnequipped` |
| **不**改 TestCharacter 数值 | 装备加成由外部属性系统消费查询 API |

### 2.6 角色属性 Stub（TestCharacter）

| 功能 | 细节要求 |
|---|---|
| 数据 | 5 项 `StatType` 基础值，Inspector 可填 |
| 显示 | 背包 UI 5 个独立 TMP（一属性一文本框） |
| 标签 | `StatDisplayUtil` 英文标签（TMP 字体策略） |

### 2.7 商店 UI（ShopUI）

| 功能 | 细节要求 |
|---|---|
| Tab | **Buy**（货架）/ **Sell**（背包列表）——面板内模式切换，非 QuickSwitch |
| 购买 | **左键点商品** → 固定位置 `ItemInfoPanel` + 栏内 **Buy**（缺货/缺钱仍显示按钮与原因） |
| 信息栏 | **不跟鼠标**；`InfoBody` 可 Scroll；**无稀有度**；一字段一 TMP |
| 墨币 | Header 显示 Ink，订阅 `WalletService.OnChanged` |
| 刷新 | 库存/钱包变化时 Rebuild 列表 |

### 2.8 背包 UI（InventoryUI）

| 功能 | 细节要求 |
|---|---|
| 列表 | 订阅 `Inventory.OnChanged` |
| 信息栏 | **左键点物品** → 点击位置弹出、**固定不跟鼠标** + Equip / Use / Unequip |
| 符文槽 | 选中槽 → 点背包符文 Equip；点已装槽 Unequip |
| Use | `TryConsumeAt(..., UseContext.Exploration)`；只 Log 效果名，不执行 |
| 属性区 | 基础值 + 装备 `statMods` 加成显示 |

### 2.9 面板互斥（StoreInventoryPanelController）

| 功能 | 细节要求 |
|---|---|
| 互斥 | **商店与背包不能同时显示** |
| 开商店 | 关背包 UI + 必要时 `ShopService.Open` |
| 关商店 | 关 ShopUI + **`ShopService.Close`（归档限购）** |
| 测试键 | 子工程 **B**=商店、**I**=背包（`Test/ItemTestController`，**勿拷主工程**）；接入后改 HUD/NPC 触发 |

### 2.10 存档（StoreSaveService）

| 功能 | 细节要求 |
|---|---|
| 键 | `inv.v1` / `equip.v1` / `wallet.v1` / `character.v1` / `shop.{shopId}.v1` |
| API | `CaptureAll` / `ApplyAll` / `RoundTripSelfTest` |
| 格式 | `JsonUtility` + `List`（不用 Dictionary） |
| **无**磁盘 I/O | 主工程存档系统调用 JSON 字符串即可 |

### 2.11 对外查询（Step13）

| API | 消费者 |
|---|---|
| `EquipmentService.GetAllStatMods()` 等 | 角色属性 / 战斗 |
| `Inventory.GetConsumables()` | 战斗管理器 |
| `Inventory.TryConsumeAt(...)` | 战斗内按格消耗 |

## §3 代码目录规范

完整树与「新脚本放哪」对照见 **[[脚本结构]]** 与工程内 `Assets/Script/StoreAndInventory/README.md`。

```
StoreAndInventory/
├── Attributes/     Inspector 特性
├── Common/         日志等基础设施
├── Catalog/        ItemDatabase 等索引
├── Services/       Inventory / Shop / Wallet / Equipment / Save / Character
├── Data/           纯数据 & SO（Item/Definitions, Effects, Shop, …）
├── UI/             Common / Shop / Inventory
├── Test/           ItemTestController
├── Future/         预留扩展说明
└── Editor/         Setup / Inspectors
```

| 禁止 | 原因 |
|---|---|
| UI 写业务规则 | 保持可测 |
| Data 引用 UGUI | 分层 |
| Service 互相反向依赖 | 见 §1 |

## §5 验收清单（功能不回退）

Play 模式下须满足：

- [ ] B 开商店，I 开背包，**不可同时开**
- [ ] 左键点商品 Buy、Sell Tab 卖、信息栏 Equip/Unequip/Use
- [ ] 商店信息栏固定位置；背包信息栏点击位置固定；字段分行、可 Scroll
- [ ] ExtQuery / SaveTest 通过（子工程 Test 场景）
- [ ] 关商店再开，符文**限购剩余**保持（runtime 归档）

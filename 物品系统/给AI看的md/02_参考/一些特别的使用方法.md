# 一些特别的使用方法

策划 / 程序配表与接 API 时的约定。完整说明见 [[系统使用手册]] §2.6–§2.7。

---

## 1. 技能描述放哪里

**skillMods 类符文**（改某技能数值，如「火球 +1 颗」）：

- 玩家能读懂的说明 → 写在物品 **`description`**
- 结构化数值 → 填 `Equipment.skillMods`（`targetId` 与将来战斗里 `skillId` 对齐）

---

## 2. GameplayEffectSO 是干什么的

Create → **MoYuan/Effect/Gameplay**，用于**独立机制效果**（被动、符箓触发、特殊规则），不是技能本体。

| 字段 | 作用 |
|---|---|
| `effectId` | 全局唯一；战斗系统执行时认这个 |
| `effectTag` | 分组 / 去重（如 `crit_no_falloff`） |
| `displayNameKey` / `descriptionKey` | UI 展示 |

**为什么要资产化？**

- 一条效果可被多物品引用（复用）
- 背包只传 SO 引用，不写战斗逻辑（解耦）
- 以后可在 SO 上扩展字段，旧物品不用重做

---

## 3. 三种「和技能/效果有关」的数据

| 配在哪 | 例子 | 本期会不会跑 |
|---|---|---|
| `Equipment.skillMods` | 火球 +1 颗 | ❌ 只聚合，战斗读 `GetAllSkillMods()` |
| `Equipment.extraEffects` | 暴击不衰减 | ❌ 战斗读 `GetAllExtraEffects()` |
| `Consumable.useEffects` | 雷电符 | ❌ `TryConsumeAt` 只返回列表 |

**技能本体**（伤害、冷却、动画）→ 将来在主工程 **SkillDefinitionSO + 战斗模块**，不在 StoreAndInventory 里。

---

## 4. 配表速查

**符箓（使用时触发）**

1. 建 `effect_xxx.asset`，填 `effectId`
2. 建 `cons_xxx`，`useEffects` 拖入，`description` 写玩家文案
3. 注册 ItemDatabase

**符文（改技能参数）**

1. 定 `skillId`（如 `fire_ball`）
2. `skillMods`：`SkillId` + `fire_ball` + `ProjectileCount` + `1`
3. `description` 写「火球术额外 +1 颗」

**符文（被动机制）**

1. 建 `effect_xxx.asset`
2. `extraEffects` 拖入
3. 战斗侧按 `effectId` 注册逻辑

# AI 速读 · 商店背包系统

> **给新 AI 的第一份文档**。读完应能回答：系统管什么、代码在哪、怎么跑通、接主工程动哪里。

---

## 1. 一句话

独立子系统：**物品数据 + 背包/商店/装备/货币运行时 + UGUI**，通过 Service 对外提供买/卖/装/卸/消耗/存档块/查询；**不执行战斗与 GameplayEffect**。

---

## 2. 三条铁律

1. **依赖方向**：`UI → Services → Catalog → Data`（Data 无 UGUI）
2. **UI 不写规则**：买/卖/装/卸只调 Service，不直接改 `InventoryData`
3. **效果只聚合不执行**：`statMods` 供 UI/主工程读；`skillMods` / `extraEffects` **本系统不跑**

## 9. 常见误区

| 误区 | 事实 |
|---|---|
| `TryConsumeAt` 会执行 effect | 只返回 `List<GameplayEffectSO>`，主工程执行 |
| `statMods` 自动改 `TestCharacter` | 仅 `InventoryUI.RefreshStats` 做 **显示**；战斗结算主工程做 |
| 拷 `Test/` 进主工程 | 不需要；会造成 B/I 与重复 Controller |
| Setup 可跳过 | 改 UI 或重复组件后应重跑 Full Setup + Save Scene |

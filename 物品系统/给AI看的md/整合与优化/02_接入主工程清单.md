## §3 场景接线步骤

### 3.1 最小运行时

1. 实例化 `Data.prefab` → 确认 `ItemDatabase` 引用全部 Item SO
2. 实例化 `Service.prefab` → 确认 SerializeField 互引（或 Awake Find）
3. 实例化/合并 `UI.prefab` → `StoreInventoryPanelController` 绑 `shopUI` / `inventoryUI`
4. **不要**挂 `ItemTestController`

### 3.2 入口改造

| 子工程 | 主工程 |
|---|---|
| B 键 `ToggleShop` | 地图 NPC / 区域 → `shopService.Open(table)` + `panel.OpenShop()` |
| I 键 `ToggleInventory` | HUD 背包按钮 → `panel.OpenInventory()` |
| 关面板 | `panel.CloseShop()` / `CloseInventory()`（会触发 Shop Close 归档） |

### 3.3 TestCharacter 替换（仅文档，本阶段不改代码）

1. 主工程角色组件提供 **基础属性**（对应 `StatType`）
2. `InventoryUI.RefreshStats` 现读 `TestCharacter.Get` + `EquipmentService.GetAllStatMods` → 改绑主角色类型（接入时改一处 UI 或抽接口）
3. `StoreSaveService` 存档块含 `SaveKeys.Character` → 主存档合并时决定保留或迁移

## §4 存档对接

```csharp
// 捕获（挂主存档保存前）
string storeJson = storeSaveService.CaptureAllJson();

// 还原（读档后）
if (!storeSaveService.ApplyAllJson(storeJson, out var err))
    Debug.LogError(err);
```

**主工程负责**：写磁盘 / 版本迁移；本系统只做内存 JSON。

## §6 版本差异备忘（主工程更新）

| 检查项 | 说明 |
|---|---|
| TextMeshPro | 子工程 UI 全 TMP；主工程需同名包 |
| Input System | 子工程 Demo 用旧 Input；主工程入口勿依赖 `ItemTestController` |
| asmdef | 避免与主工程同名程序集；必要时改 `StoreAndInventory` 程序集名 |
| API 过时 | 高版本 Unity 可能对 `FindObjectOfType` 等提示；可后续统一 |

# 修改物品系统 · 注意事项

> 改 Data / SO / id / 存档结构前必读。避免「改一行、全库资产失效」。

---

## §1 分层约束（不可破坏）

| 规则 | 违反后果 |
|---|---|
| `Data/` 不引用 `UnityEngine.UI` | 编译错误 or 分层崩溃 |
| UI 不直接改 `InventoryData.slots` | 堆叠/装备规则被 bypass |
| Service 不引用具体 UI 类型 | 无法单测、接主工程耦合 |

---

## §2 改 `ItemBase` / 子类字段

| 改动 | 连锁影响 | 建议 |
|---|---|---|
| 改 `id` 前缀规则（`equip_` 等） | 现有 SO `OnValidate` 重写 id；ShopTable、存档 JSON 断引用 | 写迁移脚本； bump 存档 version |
| 改 `idSuffix` 语义 | 资产文件名自动重命名（Editor） | Git 注意大量 rename |
| 增删 `Equipment.statMods` 结构 | UI 属性行、战斗读取 | 同步 [[系统使用手册]] §2.6 |
| 改 `Consumable.useEffects` | `TryConsumeAt` 返回列表变化 | 主工程执行器对齐 |
| 改 `maxStack` / 堆叠规则 | 旧存档 slot 可能非法 | Apply 时校验或清洗 |

---

## §3 改 `ItemStack` / 背包结构

| 改动 | 影响 |
|---|---|
| 增删 `instanceGuid` | 装备离包逻辑、存档、Shop 限购追踪 |
| 改 `InventoryData.capacity` | `0`=无限；改默认影响新档 |
| 改 slot 列表为 Dictionary 等 | **存档格式破坏性变更**，必须升 `SaveKeys.InventoryVersion` |

---

## §4 改 `EquipmentService`

| 改动 | 影响 |
|---|---|
| 改 `runeSlotCount` | UI 槽位数、存档 `EquipmentSaveData`、已有档槽位溢出 |
| 改装/卸流程 | UI `EquipmentSlotUI`、事件 `OnEquipped` |
| 在 Service 内「执行」skillMods | **越界**；应放主工程战斗模块 |

---

## §5 改 `ShopService` / ShopTable

| 改动 | 影响 |
|---|---|
| 改 `ShopTableSO.shopId` | 运行时归档 key 变；玩家限购进度丢失 |
| 改 `stock` 语义 | 装备限购 UI `StockText` |
| 删 `Close()` 归档 | 关 UI 丢限购（已 fix，勿回退） |

---

## §6 改存档

| 项 | 说明 |
|---|---|
| `StoreSaveService.BundleVersion` | 整包格式变更时 +1 |
| 各 chunk `version` | 单块 JSON 结构变更时 +1 |
| 新增 chunk | 旧档无此块 → Apply 跳过即可 |
| 删 chunk | 旧档数据丢失 |

Apply 失败时看 `StoreSaveService.ApplyAll` 返回的 `error` 字符串。

---

| 改动 | 影响 |

## §8 安全修改流程（建议）

1. 改前查 [[系统功能与工程规范]] 对应 §2 条目
2. 若动存档 → 升 version + 写迁移或清档说明
3. 若动 id → 批量重导 SO + 更新 ShopTable
4. 改后：SampleScene Full Setup → §5 验收清单
5. 更新 [[工程清理与优化记录]]

## §7 日志符号

在 **Project Settings → Player → Scripting Define Symbols** 添加：

```
DEBUG_STORE_INVENTORY
```

未定义时：Buy/Sell/Add/Take 等成功路径 **不输出** Info 日志；Warning/Error 仍输出。

## §8 第二轮脚本大整理（2026-05-26）

> 原则不变：**不改变 §5 验收清单可观察行为**。

### 8.1 目录合并

| 变更 | 说明 |
|---|---|
| `Store/ShopService.cs` → `Core/ShopService.cs` | 删除空 `Store/` 文件夹；服务统一在 Core |
| `Inventory/Inventory.cs` → `Core/Inventory.cs` | 删除空 `Inventory/` 文件夹；与 `Data/Inventory/` 职责分离 |

### 8.2 删除遗留 API

`Inventory.cs` 移除 `[Obsolete]` 方法：`Add` / `Remove` / `RemoveLast` / `GetAll` / `LogAll`（工程内无调用方）。

### 8.3 程序集拆分

| 文件 | 作用 |
|---|---|
| `StoreAndInventory.asmdef` | Runtime（Core + Data + UI + Test） |
| `Editor/StoreAndInventory.Editor.asmdef` | Editor 专用，引用 Runtime |

### 8.4 新增

| 文件 | 用途 |
|---|---|
| `Editor/SceneCleanupUtility.cs` | 删 `SAI/Shop` Missing Script、Canvas 根 duplicate Tooltip/ContextMenu |
| `StoreAndInventory/README.md` | 工程内目录与命名约定 |

`Setup Full Store Loop` 现自动调用场景清理。

### 8.5 仍未做（留后续）

| 项 | 原因 |
|---|---|
| Shop/Store 类型重命名（如 `Equipment` → `GearItem`） | 影响 SO 资产与存档 id，需主工程接入时评估 |

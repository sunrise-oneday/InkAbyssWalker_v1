# 2D类银河城探索与双状态机回合制战斗系统——项目架构与功能设计白皮书

本白皮书旨在对当前项目的技术架构、核心系统及各模块（Class）的职责分工进行深度梳理与总结。项目完美融合了 **2D类银河城物理探索** 与 **时机招架回合制战斗**（机制参考《33号远征队》与《朝露：境界旅程》），底层采用了高度解耦、强类型安全、零运行时GC、事件驱动及观察者模式等现代游戏工业级设计范式。

---

## 一、 项目核心设计思想

### 1. 双状态机（Dual FSM）与脑区隔离设计
这是本项目最具含金量的架构设计。主角物体拥有两个完全独立的状态机：
*   **探索状态机（Exploration FSM，所有者为 `PlayerController`）**：负责管理大地图上的物理移动、跳跃、冲刺、坠落及攻击。
*   **战斗状态机（Battle FSM，所有者为 `PlayerBattleEntity`）**：负责管理战斗中的待机、施法、时机招架、完美闪避、奥义释放及死亡。
*   **无缝脑区切换**：进入战斗时，大地图控制器 `PlayerController` 脚本组件被禁用，大地图物理与跑跳按键信号全自动归零（大地图脑区冬眠）；同时激活 `PlayerBattleEntity`（战斗脑区唤醒），开始独立运行战斗状态机。战斗结束时反向切换。两套状态机在内存中常驻且物理隔离，杜绝了变量堆积和状态冲突。

### 2. 纯事件驱动与强类型输入缓存
彻底告别了“每帧轮询按键”导致的样板代码地狱（Boilerplate Hell）与漏键Bug。输入系统通过 `InputManager` 和专职的 `Reader` 将按键信号转换为事件进行广播。
*   **输入时间戳缓存（Input Timestamp Buffer）**：对于单次触发类按键（跳跃、闪避、招架），不记录 `bool` 值，而是记录按下时的系统时间戳（`Time.time`）。
*   **自动生命期衰减**：规定输入有效期为 150 毫秒（`InputBufferTime = 0.15f`）。进入判定状态时，只需比对时间差。如果超过 150 毫秒则自动过期失效，无需在其他状态里写任何繁琐的手动清空代码。同时，该机制自动为游戏带来了极佳的手感功能——**预输入（Input Buffering）**。

### 3. 数据与表现分离（MVC 与观察者模式）
角色属性、Buff 状态完全收拢在 `CharacterStats.cs` 中。UI 表现层（`EntityHUD`、`BattleUIController`）作为观察者，通过订阅属性组件的事件（如 `OnHPChanged`、`OnBuffsChanged`）实现**自响应式刷新**。数值修改与 UI 渲染完全解耦，大幅降低了中后期的维护成本。

## 二、 核心功能实现清单

### 1. 2D类银河城物理探索
*   基于 `Rigidbody2D` 的精准运动控制与惯性衰减。
*   **单点物理悬崖探路（Ledge Check）**：利用 `transform.localScale.x *= -1` 进行镜像翻转，使子物体检测点自动同步晃动到身体前方鞋底处，实现单检测点的高效悬崖停步与待机。
*   **二段跳与空中微调**：通过空中状态基类统一管理，解锁后可动态获得空中二段跳资格。

### 2. 无缝叠加场景战斗过渡（Additive Scene Loading）
*   碰怪或被偷袭时，大地图瞬间挂起，异步叠加加载 `BattleScene`。
*   **刚体安全传送**：利用 `rb.position` 与 `Physics2D.SyncTransforms()` 强行打穿物理引擎缓存，将主角物理传送至远处隔离区的战斗擂台 `(2000, 2000)`。
*   **自动双重卸载**：战斗结束，卸载 `BattleScene`，临时克隆的队友、怪物等物体在瞬间被干净释放。主角一秒被传送回大地图原位，大地图相机和物理无缝恢复。

### 3. 多形态切换与共享资源池
*   **共享队伍资源**：全队共享公共的 AP（行动点）和 MP（蓝量）资源池。
*   **动态卡牌渲染**：切换形态（消耗 AP）时，主 UGUI 技能面板不仅会播放变身过渡动画，还会全自动读取该新形态下配置的技能，动态将其刷入下方的 UGUI 技能卡牌按钮并重新绑定点击事件，完全免去手动拖拽。

### 4. 精准时机招架与闪避系统（QTE 防守机制）
*   **时机预警（闪红提示）**：怪物抬手劈砍前 `0.2s`，怪物身体瞬间闪烁红光（`TriggerParryIndicator`），为玩家提供最直观的按键视觉哨兵。
*   **完美招架（Perfect Parry）**：在伤害落点前 `120` 毫秒内按下空格键。免除 100% 伤害与削韧，触发震屏、卡肉和翡翠绿闪光表现。
*   **防乱按惩罚**：设计了 350 毫秒的招架/闪避冷却时间。任何狂敲键盘的玩家都会在瞬间被冰冻惩罚，强行判定招架失败。
*   **完美闪避（Perfect Dodge）**：按 Shift 触发。在落点前 `120` 毫秒内触发会启动**“魔女时间（子弹时间）”慢动作特写（画面放慢 5 倍）**，全队共享 AP +1（单回合上限一次）。
*   **多段无痕承接**：闪步（0.25s）结束后，角色会自动端起武器退回招架姿势，玩家可继续使用键盘应付接下来的连斩。

### 5. 状态异常、叠层与元素反应系统
*   **伤害拦截钩子（OnBeforeTakeDamage）**：在计算实际伤害前调用。状态本身可以作为拦截器，极易实现“免疫护盾”、“免伤盾”或“流血伤害加深”。
*   **元素反应机制（融化/蒸发）**：怪物身上挂着火，玩家施加冰，底层自动引爆“融化（Melt）”。火/冰状态消失，怪物破防条遭到重创，且套上高额易伤 Buff。
*   **叠层威力增幅**：给敌人重复上同名元素，不生成新图标，而是增加持续时间（上限 5）并**叠加层数（最大 3 层）**。发生元素反应时，**反应的伤害和削韧值直接乘以当前的叠层数**，层数越高，爆发越恐怖！

### 6. 大地图篝火自动存档
*   挂载 `CheckpointTrigger` 组件，大地图碰撞即可激活存档，角色状态完全充满。
*   **防呆写盘防火墙**：由于战斗擂台在 `(2000, 2000)` 的隔离区。`SaveManager` 的写入端加装了坐标过滤器，一旦检测到大于 `1500` 的脏坐标，直接强制拦截并拒绝写盘，确保存档绝对安全。

## 四、 核心工作流深层解析

### 1. 从“探索”切入“战斗”场景
1.  玩家在 `ExploreScene` 操控主角。主角行走至悬崖边，`PlayerController` 通过 `CheckGroundAhead()` 探测到前方有路，但在 `0.4` 秒前（根据 `ledgeCheckPoint`）小怪 `OverworldEnemy` 也在朝玩家巡逻。
2.  小怪进入警觉范围 `detectRange`，切入 `EnemyChaseState` 开始全速追击玩家。
3.  小怪追上玩家（进入 `attackRange`），切入 `EnemyAttackState` 进行飞扑斩击。
4.  如果在 0.45 秒内玩家没有按 Shift 闪避，怪物扑中玩家，调用 `BattleManager.Instance.StartBattle(groupIndex, isPreemptive: false)` [3]。
5.  `BattleManager` 立即执行：
    *   暂存主角当前的碰怪大地图坐标。
    *   禁用 `PlayerController`（大地图跑跳、物理位移瞬间绝对静止），但**不禁用其 GameObject**，使其 stateMachine 的 Update 在后台安全冬眠。
    *   禁用 `GamePlayer` 动作表，开启 `Battle` 战斗动作表（鼠标显示并解锁） [2]。
    *   异步叠加加载 `BattleScene` [3]。
6.  加载完毕后，`BattleManager` 根据 `groupIndex` 动态在战斗场景的 `EnemyBattleSpawn` 上克隆对应的怪物预制体，并将主角**刚体物理传送**到 `PlayerBattleSpawn` 位置，调用 `Physics2D.SyncTransforms()` 强行物理对齐 [3]。
7.  启用玩家的 `PlayerBattleEntity` 战斗组件，玩家和怪物战斗状态机同时启动 [2]。由于被怪物偷袭，开局直接调用 `EnterEnemyTurn()` 进入敌人回合，玩家瞬间被强行推入 `PlayerParryState` 架起盾牌，准备战斗 [3]！

### 2. 招架/闪避 QTE 防守时序
1.  怪物回合开始，在 2.0 秒的对峙时间后 [3]，小怪 1 的状态机切入 `EnemyAttackState` 开始出招。
2.  在劈砍动作落下的前 `0.2` 秒，怪物动画轨道触发 **`TriggerParryIndicator`** 事件：
    *   怪物全身瞬间**闪烁红光**警告，作为玩家按键的“视觉哨兵”！
3.  玩家集中注意力：
    *   如果玩家处于**格挡形态（2）**并按下了空格键：`parryPressTime` 记录为当前 `Time.time` [1, 2]。
    *   当怪物动画播放到劈中你的这一帧时，动画轨道自动触发 **`TriggerDamage(int hitIndex)`** 伤害点 [1]。
    *   `EnemyBattleEntity` 接收事件，并直接通知 `BattleManager.EvaluateParryAndApplyDamage(hitIndex, seq)` [1]。
    *   战斗管理器计算时间差：`hitTime - parryPressTime`。如果在 120 毫秒内，判定为 **“完美格挡（Perfect Parry）”** [1]：
        *   免除 100% 伤害和削韧。玩家身体闪烁**亮蓝色防护光芒** [1]。
        *   游戏时间流速慢放至 0.05 倍（**卡肉顿挫 0.06 秒**），战斗摄像机**猛烈震动 0.2 秒**。
4.  当怪物播放完它的攻击动画，在最后一帧触发 **`TriggerAttackFinished`** 事件 [1]：
    *   怪物自己退回待机状态。
    *   如果玩家成功完美招架了怪物本轮的所有多段劈砍（`allPerfectParriesInCurrentAttack` 保持为 `true`）：
        *   回合推进**暂时挂起暂停**！玩家状态机强制切入 `PlayerCounterAttackState` 腾空跃起挥出大招反击 [2]！
        *   反击动画播放完毕（自动读取实际时长），对怪物造成巨额伤害和破防值 [2]。
        *   动作结束，玩家退回待机，主动调用 `ProceedEnemyTurn()` 恢复回合运转，推进下一只怪出手或进入玩家回合 [2, 3]！

### 3. 战斗结束与重生
1.  每次伤害结算后，`BattleManager` 都会执行 `CheckBattleOver()` 进行存活检测。
2.  **如果胜利（怪物全灭）**：
    *   弹出战斗胜利 UI 结算面板，停留 3 秒给玩家观赏 [3]。
    *   禁用 `Battle` 动作表，启用 `GamePlayer` 动作表（鼠标重新锁定隐藏） [2]。
    *   异步卸载 `BattleScene`（克隆的怪物和队友随场景一并被干净销毁释放） [3]。
    *   大地图主相机恢复，主角物理激活并被传送回大地图碰怪原位，物理手感完好如初 [3]。
    *   大地图原地自动调用 `SaveManager.Instance.SaveCheckpoint(position)` 存档。
3.  **如果败北（玩家死亡）**：
    *   玩家状态机切入 `PlayerBattleDieState` 播放倒地死亡动画并关闭物理碰撞。
    *   弹出战败 UI 结算面板 [3]。
    *   由于需要刷新怪物，**直接重新异步加载整个大地图 `ExploreScene`（Single 模式）**。整个大地图上刚才打碎的瓶子、击杀的怪物全部在底层实现最完美、最安全的物理重置复活！
    *   大地图重新加载完毕，`SaveManager` 读取最后的篝火存档点数据，将满血复活的全新主角**精准物理传送到最后一个篝火点位置**，玩家在此醒来，惩罚机制完满闭环！
